## 七、Universal Execution Chain（所有模型共享的通用序列）

精读全部资料后发现，**所有 ICT 入场模型共享同一个 8 步通用序列**：

```
1. 确定 HTF Bias（Weekly/Daily IPDA 20D 范围 + CISD）
2. 识别 Draw on Liquidity（旧 NWOG/周低/PDH/PDL/EQL/EQH）
3. 等待流动性扫荡（BSL/SSL 被扫，对手方流动性被清除）
4. 确认位移（MSS/CISD 发生，创建 FVG/IFVG/Breaker/OTE 区域）
5. 时间窗口对齐（必须在 KZ 或 Macro 内）
6. 在 PDA 入场（← 这一步是各模型的差异点）
7. 目标对手方流动性（不是固定 R:R，而是流动性池）
8. 一单即走（单 Session 单笔，盈亏后停止，最大 2% 风险）
```

**步骤 6 是唯一的差异点**——2022 用 FVG，Unicorn 用 FVG+Breaker 重叠，OTE 用 62-79% Fib 区域，等等。其他 7 步完全相同。

这意味着 Michael 的分析引擎应该：
- **步骤 1-5 + 7-8 是固定工作流**（Skill 模块覆盖）
- **步骤 6 是模型选择引擎**（基于 Market State 条件匹配）

## 八、ICT 2022 Mentorship Notes 补充发现

> 来源：@TanjaTrades 笔记，覆盖 2022 Mentorship Ep.2-41（43 页）

### 8.1 对已有框架的补充与精化

| 概念 | 已有理解 | 2022 Notes 补充 |
|------|----------|----------------|
| **FVG 入场规则** | 回撤到 FVG 入场 | **三触规则**：FVG 被有效触及 2 次后，第 3 次不应再指望它守住 |
| **FVG 有效性前提** | 任何位移后的 FVG | **仅在重大位点**（swing H/L 被扫后）的 FVG 才有效，不要找随机 FVG |
| **双 FVG 场景** | 未明确 | 预期价格先探更深的 FVG，等回撤到浅 FVG 时入场（用浅的做入场点） |
| **OB 有效性** | 位移前的最后反向 K 线 | **必须后接 imbalance/displacement**——没有 imbalance 的反向 K 线不是 OB |
| **OB 入场位** | OB 区域入场 | **以 OB 开盘价 + 3 ticks 挂单**；Mean Threshold (50%) 是二级位 |
| **Gold Standard 入场** | 未定义 | **FVG + Order Block + OTE after short-term MSS** = 最高概率组合 |
| **Midnight Open** | 作为参考价 | **框架整个日内范围**的锚点；London (2-5AM) 的框架起点 |
| **8:30 AM Open** | 作为参考价 | 如果看空且 8:30 Open < Midnight Open，用更低者作为 Judas Swing 最低门槛；**两者都未被突破 = 极度看空** |
| **IOF 实操** | OB 作为支撑/阻力 | 看多时：**下跌 K 线的顶部不应被突破回去**（机构在卖）；看空时：**上涨 K 线的顶部不应被反弹打穿** |
| **IRL vs ERL 出场** | 笼统的目标设定 | **IRL = 部分止盈**（前序 FVG 或 50% fib）；**ERL = 全仓止盈**（日内低/高之外） |

### 8.2 新增实操规则（之前未覆盖）

| 规则 | 内容 | 重要度 |
|------|------|--------|
| **最大交易次数** | 早盘最多 2 笔 + 午后最多 2 笔 = 日内 ≤4 笔 | 🔴高 |
| **典型持仓时长** | 90 分钟到 2 小时 | 🟡中 |
| **最佳 LTF 时间框架** | 指数用 1/2/3 分钟图（不用 5 分钟），3 分钟图最佳 | 🔴高 |
| **大行情次日** | 大行情日后的次日早盘**不要实盘交易**，用模拟盘 | 🟡中 |
| **新闻日过滤** | 只在有中/高影响新闻的日子交易——低影响新闻日 = 不交易 | 🔴高 |
| **FOMC/NFP 规则** | 如果本周已有盈利，FOMC/NFP 当日不交易 | 🟡中 |
| **午餐唯一例外** | 快速市场中的**回撤**（非盘整）期间可以交易午餐时段 | 🟡中 |
| **亏损后缩量** | 亏损后下一笔必须**减小仓位**，不是保持不变或加大 | 🔴高 |
| **风险起步** | 新手期风险 < 1%（ICT 本人用 3.5%） | 🔴高 |
| **止损设置** | 放在 FVG 的 Candle 1 或 Candle 2 上方（做空）；大 imbalance 时放 imbalance 顶部 | 🟡中 |
| **加仓（Pyramiding）** | 首次 3 手→后续 FVG 加 2 手→再加 1 手，沿途在后续 FVG 位置加仓 | 🟡中 |
| **Close Proximity Entry** | 错过理想入场点时，可在附近或下一个 BOS+FVG 处入场（只要 R:R 仍然合理） | 🟡中 |
| **Bias 五日视野** | 日线 bias 的预测范围限制在 5 个交易日内——不要试图预测更远 | 🔴高 |
| **Bias 失效** | 如果 ITH 被向上突破（看空时），立刻承认错误退出——不要硬扛 | 🔴高 |

### 8.5 与多品种分析的关联

2022 Notes 明确了多品种分析的实操方法：
- **ES 看入场，NQ 做交易**：因为 ES/NQ 同向但 ES 更稳定，用 ES 确认入场时机
- **YM 做背离确认**：YM 没有跟随 ES/NQ 创新低 = 卖方流动性已被扫，看多信号
- **背离≠独立信号**：单独看背离**毫无意义**，必须有叙事支撑（ICT 曾因只看背离而爆仓）
- **DXY 宏观过滤**：美元上涨 = 风险关闭，其他资产下跌

这些规则应纳入 `bias.md` 和 `iof_confirmation.md` Skill 中。
